# Sales Analysis Dashboard – Power BI Project


## Objective

* To monitor total sales and number of orders
* To analyze sales distribution across regions and products
* To evaluate employee performance
* To present data through interactive visualizations

### Sales Data (January, February, March)

* Order ID
* Customer Name
* Employee ID
* Order Date
* Product
* Quantity
* Region
* Revenue
* Cost

### Employees

* Employee ID
* Name
* Department
* Region

## Data Model

* Sales tables are connected to the Employees table using EmployeeID
* Relationship type: Many-to-One (* → 1)

## Data Cleaning (Power Query)

* Renamed columns for clarity
* Converted data types (Text, Whole Number, Decimal)
* Removed errors and empty values
* Ensured unique Order IDs

## Key Insights

* Identified high-performing regions
* Compared product-level sales performance
* Evaluated employee contributions to revenue
* Observed overall sales patterns
